内置 agent 不绑定任何一家厂商 —— **由你来选谁驱动它**，切换只需一条命令。如果你是从
[概览](/zh/agent/overview/)一路看过来的，那你已经知道：**没设模型之前 agent 不会醒**。这一页讲的就是
怎么做：ref 的格式、接入了哪些 provider、为什么有的被排除在外、两个引擎，以及确切的命令。

## ref

一个模型用 `provider/model` 这样的 **ref** 来命名。斜杠前那段挑选 **provider**（以及由哪个引擎来跑它），
斜杠后那段是 **模型 id**，原封不动地传给该厂商的 API —— 所以一个模型发布当天就能用，无需更新 PhysiClaw。
例如：

```text
anthropic/claude-sonnet-4-6
openai/gpt-5
qwen/qwen3-max
```

这一个 ref 就选定了整条技术栈。它存在 `~/.physiclaw/config.toml` 的 `[agent] model` 里，或用
`PHYSICLAW_MODEL` 环境变量设置。两者都设了时，环境变量胜出。（完整的优先级规则见
[配置](/zh/reference/configuration/)。）

:::note[没有默认模型]
没有内置默认值。内置 agent 运行时会一直沉睡，直到设了一个 ref —— 在你执行
`physiclaw models key <provider>` 再执行 `physiclaw models use <provider/model>`（或导出
`PHYSICLAW_MODEL`）之前，**两种**驱动模式都会高声报错。一个不带模型、直接跑的 `physiclaw server`
照样能用于硬件设置和手动调用 MCP 工具；只有 agent 起不来。
:::

## 支持的 provider

PhysiClaw 自带的进程内引擎透过五家厂商各自的 chat API 与它们对话，外加一个列在册上但拒绝运行的占位 ——
注册表里共六个 provider id —— 再加上第七个会调起 Anthropic `claude` CLI 的 ref：

| Provider id | 厂商 / 模型 | API key |
| --- | --- | --- |
| `anthropic` | Claude，透过 Anthropic 官方 API（`/v1/messages`） | `ANTHROPIC_API_KEY` |
| `openai` | GPT / 推理模型（如 `gpt-5`、`o4-mini`） | `OPENAI_API_KEY` |
| `google` | Gemini，透过 Google 的 OpenAI 兼容端点 | `GOOGLE_API_KEY` |
| `moonshot` | Kimi（K2） | `MOONSHOT_API_KEY` |
| `qwen` | Qwen，透过阿里 DashScope | `QWEN_API_KEY` / `DASHSCOPE_API_KEY` |
| `deepseek` | 列在册上但是有意为之的占位 —— 见下文 | `DEEPSEEK_API_KEY` |
| `claude-code` | 以子进程方式运行 `claude` CLI（复用 `anthropic` 的模型清单 + key） | `ANTHROPIC_API_KEY` |

每个 key 都从环境变量解析 —— 默认是大写的 `<PROVIDER>_API_KEY` —— **或者**来自 `config.toml` 里的
`[provider] <id>_api_key`。两者都设了时，环境变量胜出。注意 `anthropic` 和 `claude-code` 都读
`ANTHROPIC_API_KEY`。

:::caution[为什么 DeepSeek 是占位]
`deepseek` 在注册表里，但它会在启动时抛出一个清晰的错误，而不是悄悄地盲开。它的 chat 端点 **只支持纯文本**，
而 PhysiClaw **每次 `peek` 都需要视觉**才能读懂屏幕。一条通则：PhysiClaw 要求一个既能 **看** 又能 **推理**
的模型，看不见的模型会高声报错而不是瞎猜。（出于同样的推理要求，那些默认把思考预算关掉的 Gemini 变体在
接通之前也被排除 —— 这是一个更温和的、次要的情形。）
:::

## 两个引擎，一个 ref

provider id 同时也决定了模型*怎么*跑。你不必直接管理引擎 —— 挑了 ref，就挑了引擎：

- **`anthropic/...`** 透过 **Anthropic 官方 API**（`/v1/messages`）运行 Claude，跑在 PhysiClaw
  自带的进程内引擎里 —— 即原生工具调用循环。
- **`claude-code/...`** 以 **子进程方式运行 Anthropic 的 `claude` CLI**。它的模型 id 和 API key 都是
  Anthropic 的 —— CLI 对的是同一个 API —— 所以 `claude-code` 复用 `anthropic` 的模型清单和
  `ANTHROPIC_API_KEY`。它*不*在进程内注册表里。
- **其余所有**（`openai`、`google`、`moonshot`、`qwen`）都跑在进程内引擎里，瞄准对应厂商的 API。

这一页唯一容易搞混的点：`anthropic/...` 是透过进程内引擎走官方 API；`claude-code/...` 是另起一个
`claude` CLI 子进程。同一家厂商，两种不同的跑法。

想知道进程内循环每一轮到底做了什么，见[引擎内部](/zh/agent/inside-the-engine/)。

## 设置与切换模型

`physiclaw models` 这组命令是入口。PhysiClaw **不内置任何经过挑选的模型列表** —— 它去拉取每家厂商实时的
`/v1/models` 并缓存下来，所以发现到的结果就是事实来源。`models use` 会拿你的 ref 对着缓存做校验（打错字
会立刻失败），而 `models key` 在你设置 key 时会自动触发一次发现。

<Steps>

1. **设置某个 provider 的 API key。** 省略值就会提示输入（让密钥不进 shell 历史）。写入 key 会立刻拉取
   该厂商的实时模型列表，于是你马上就能挑一个。

   ```bash
   physiclaw models key anthropic
   ```

2. **看看有哪些可用。** 列出某个 provider 发现到的模型（不带参数则列出全部）。

   ```bash
   physiclaw models list anthropic
   ```

3. **挑选当前模型**，用一个精确的 `provider/model` ref。它会对着发现缓存做校验，所以打错字会立刻失败。
   （`set` 是 `use` 的别名。）

   ```bash
   physiclaw models use anthropic/claude-sonnet-4-6
   ```

4. **重启 server** 以生效 —— 运行中的 agent 在启动时读取 `[agent] model`，而写入配置时会打印一条
   *Restart `physiclaw server` to apply* 的提醒。

   ```bash
   physiclaw server
   ```

</Steps>

裸跑 `physiclaw models` 会打印当前状态 —— 当前 ref、它来自哪里（环境变量还是配置文件），以及该 provider
的 key 是否已设置（已脱敏）：

```text
Active model
✓ anthropic/claude-sonnet-4-6  (from config.toml [agent] model)
  anthropic api key: ********  [ANTHROPIC_API_KEY env]
```

### 其余命令

| 命令 | 作用 |
| --- | --- |
| `physiclaw models` | 显示当前 ref、它的来源，以及当前 provider 的 key 状态 |
| `physiclaw models list [provider]` | 列出发现到的模型（不带参数则列出每个 provider） |
| `physiclaw models use <provider/model>` | 切换当前模型（别名：`set`） |
| `physiclaw models key <provider> [value]` | 设置某个 provider 的 API key（省略值则提示输入；自动发现） |
| `physiclaw models keys` | 显示每个 provider 的 key 状态，已脱敏，并附来源 |
| `physiclaw models discover <provider>` | 重新拉取某厂商的实时模型列表 |

:::tip[有新版发布了？]
`models key` 会自动发现，所以你很少需要手动调 `discover`。当某厂商发布了一个新模型，而你想让它在你能
`models use` 之前就先出现在 `models list` 里时，再用它。
:::

还没安装？从[安装](/zh/set-up/install/)开始。完整的命令清单见 [CLI 参考](/zh/reference/cli/)。
