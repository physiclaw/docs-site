## What to expect

<CardGrid>
  <Card title="≈ $112" icon="seti:shell">
    Off-the-shelf parts. The arm is a consumer pen plotter; the camera is an ordinary USB webcam.
  </Card>
  <Card title="≈ an afternoon" icon="seti:clock">
    Assembly is mechanical — no soldering, no custom boards. Calibration is a guided wizard.
  </Card>
  <Card title="No special skills" icon="approve">
    If you can follow a wiring diagram and clamp a gooseneck, you can build one.
  </Card>
  <Card title="macOS today" icon="apple">
    macOS is the supported platform; Windows works for most of the flow. (Serial ports differ.)
  </Card>
</CardGrid>

## The road ahead

The rest of the docs are this build, in order. Each step hands off cleanly to the next:

<Steps>

1. **[Build the hardware](/en/build/bill-of-materials/)** — order the parts, then assemble the
   arm, stylus, camera, and phone holder.

2. **[Install the software](/en/set-up/install/)** — one command installs the `physiclaw` CLI;
   `physiclaw doctor` checks your machine.

3. **[Prepare the phone](/en/set-up/prepare-the-phone/)** — turn on AssistiveTouch and add the
   three iOS Shortcuts. Five minutes, once.

4. **[Calibrate](/en/set-up/calibrate/)** — a guided wizard teaches the arm and camera exactly
   where the screen is, so taps land where the agent aims.

5. **[Run your first task](/en/run/first-task/)** — connect your agent and give it a goal in
   plain language. Watch it look, decide, and tap.

</Steps>
