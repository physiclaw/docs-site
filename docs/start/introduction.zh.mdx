你见过 AI 助理在屏幕上点来点去。**PhysiClaw 让它去碰一块真正的屏幕。**

PhysiClaw 是一台小巧的桌面机器人，它给 AI 助理一副*实体*的身体来操作手机。一台相机俯视屏幕；一支装在机械臂上的电容触控笔伸出去，点在玻璃上。助理读屏幕上的内容，决定怎么做，机械臂就照做 —— 跟人操作手机是同一套循环，只不过用相机当眼睛、用触控笔当手指。一句话概括：**一个长了手指的 AI 助理。**

软件以 `physiclaw` 包发布（Python 3.12+，主打 macOS，MIT 许可）。对手机来说，这支触控笔和指尖毫无区别 —— 所以*任何* app 都能用，iOS 或 Android 都行，每个 app 都不用单独配置。

:::tip[从 OpenClaw 过来的？]
OpenClaw 给了助理一双*软件里*的手 —— 一个 LLM 大脑，通过它已经身处其中的那块屏幕来驱动各种 app。PhysiClaw 让那半边**大脑**保持熟悉：还是 LLM/MCP 助理，自带模型、自托管、同样的安全姿态。**新增**的是那半边**身体** —— 一套真实的装置，带运动控制、校准、相机视觉，以及在一台它从未登录过的手机上做出的触控笔手势。思路是一样的 —— 让助理用人类在用的界面 —— 只是从桌面推到了玻璃上。
:::

## 它解决的问题

那些号称「能用你手机」的软件助理，靠的无非三样东西 —— API 与 OAuth、无障碍接口，或者越狱。每一样都是一堵墙：要么得逐个对接，要么挂的钩子会被检测、被搞坏，要么得 root 一台设备。PhysiClaw 干脆完全不碰软件栈，绕过了这三样 —— 唯一抵达手机的东西就是一个触控笔尖，所以没什么要对接的，没什么能被检测的，也没什么要越狱的。[为什么需要一个身体？](/zh/start/why-a-body/)把这个理由讲透。

## 它如何工作，四个动作讲完

这是一个*经过验证*的循环，不是什么遥控魔术。它每次都按同样的方式运行：

1. **看。** 俯拍相机给屏幕拍照。设备本地的视觉模型（文字用 RapidOCR，图标用 OmniParser）把找到的每个元素都框出范围、打上标签，每个都带一个 *bbox* —— 一个以屏幕 0–1 比例表示的矩形。
2. **决策。** 助理读这份标注好的清单，挑一个目标框，再挑一个手势：tap、swipe、长按。
3. **移动并触碰。** 机械臂把触控笔送到 bbox 中心，笔尖落下完成一次真实的电容触碰，然后抬起，停到屏幕外。
4. **检查。** 相机再看一眼。屏幕有没有按预期变化？变了就进行下一个动作；没变就再试一次。

因为每一步都以*查看结果*收尾，所以意外 —— 弹窗、广告、加载慢 —— 不过是要去应对的新状态，而不会让脚本崩掉。
[它如何工作](/zh/start/how-it-works/)详细走一遍完整的循环。

## 助理驱动它的两种方式

要给这套装置接上一个大脑，有两种方式，[内置助理总览](/zh/agent/overview/)里有完整说明：

- **自带 MCP 客户端。** 像 OpenClaw 或 Claude Desktop 这样的外部客户端连到服务器的 `/mcp` 端点，直接驱动这套装置。
- **内置助理运行时。** PhysiClaw 自带的引擎替你跑「看→决策→移动→检查」这个循环。

## 手机上到底装了什么？

「什么都不装」这话*差不多*是真的，而这点差别很要紧 —— 所以直说了吧：

:::note[老实版]
**没有 App Store 上的 app，没有开发者模式，也没有越狱。** 但你确实要把手机一次性准备好：打开 **AssistiveTouch**，添加三个简短的 **iOS Shortcuts**（一次 tap、一张像素级精准的 screenshot、以及剪贴板同步），并在使用期间让浏览器里一个小小的**桥接页面**保持打开。它依旧是一台你花五分钟设置过的普通手机。
[准备手机](/zh/set-up/prepare-the-phone/)讲清楚到底装什么、为什么。
:::

请用一台一次性 / 备用手机 —— 运行时的 `unlock_phone` 会输入一个写死的密码（`111111`），所以这台工具手机最好是你不介意暴露的那种。完整的安全姿态见 [安全](/zh/safety/)。

## 它适合谁，你要承诺投入什么

PhysiClaw 是给爱动手的构建者和折腾党的 —— 学机器人的人，以及所有愿意亲手组装一台机器、照着接线图接线的人。这是一个**从零搭建**的项目，不是买来一台现成绘图仪、装上一支笔就完事：

- X/Y 运动是一套手工搭建的 **CoreXY** 龙门架（双电机 X/Y 平台）。触控笔靠一个**电磁铁**落到屏幕上 —— 这是 Z 轴上的推杆执行器，不是舵机也不是笔。
- 零件成本大约 **$145 美元**（在中国约 ¥1046；美元/欧元市场会更高）。
- 预计要**做一些焊接**来接线，还要做一次性的**固件刷写**（FluidNC）。
- 这是一个**跨数天的项目**：采购、切割、3D 打印零件（供应商备货要好几天），然后大约 **2–3 小时的组装**，接着刷固件，最后是一段反复的**校准与调试**阶段，把点击调到分毫不差。

这些都不算稀奇 —— 对喜欢动手的人来说，这是一个真实但完全够得着的「周末加一点」的项目。完整的成本与零件清单在 [采购零件](/zh/hardware/sourcing/)，搭建本身从 [组装](/zh/hardware/assembly/) 开始。

:::note[平台]
目前支持安装的是 macOS；整个流程的大部分在 Windows 上也能跑。
:::
