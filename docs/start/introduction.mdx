You've seen an AI agent click around a screen. **PhysiClaw lets it touch a real one.**

PhysiClaw is a small desktop robot that gives an AI agent a *physical* body to operate a
phone. A single camera looks down at the screen; a capacitive stylus on a moving arm reaches
out and taps the glass. The agent reads what's on screen, decides what to do, and the arm does
it — the same loop a person runs, just with a camera for an eye and a stylus for a finger. In
one line: **an AI agent with a finger.**

The software ships as the `physiclaw` package (Python 3.12+, macOS-focused, MIT). To the phone,
the stylus is indistinguishable from a fingertip — so *any* app works, iOS or Android, with no
per-app setup.

:::tip[Coming from OpenClaw?]
OpenClaw gave an agent hands *inside software* — an LLM brain that drives apps through the
screen it already lives on. PhysiClaw keeps that **brain** half familiar: an LLM/MCP agent,
bring-your-own-model, self-hosting, the same safety posture. What's **new** is the **body** —
a real rig with motion control, calibration, camera vision, and stylus gestures on a phone it
was never logged into. Same idea — let the agent use the interface a human uses — pushed off
the desktop and onto the glass.
:::

## The problem it solves

Software agents that "use your phone" lean on one of three things — APIs and OAuth,
accessibility hooks, or jailbreaks. Each is a wall: integrations to build, hooks that get
detected and broken, or a device you have to root. PhysiClaw sidesteps all three by not
touching the software stack at all — the only thing reaching the phone is a stylus tip, so
there is nothing to integrate, nothing to detect, and nothing to jailbreak.
[Why a body?](/en/start/why-a-body/) makes the full case.

## How it works, in four moves

This is a *verified* loop, not magic teleoperation. It runs the same way every time:

1. **Look.** The overhead camera photographs the screen. On-device vision (RapidOCR for text,
   OmniParser for icons) boxes and labels every element it finds, each with a *bbox* — a
   rectangle given in 0–1 fractions of the screen.
2. **Decide.** The agent reads that annotated listing and picks a target box plus a gesture:
   tap, swipe, long-press.
3. **Move & touch.** The arm drives the stylus to the bbox center; the tip drops to register a
   real capacitive touch, then lifts and parks off-screen.
4. **Check.** The camera looks again. Did the screen change the way it should? If yes, on to
   the next action; if not, retry.

Because every step ends by *looking at the result*, a surprise — a popup, an ad, a slow load —
is just the new state to react to, not a script that's fallen apart.
[How it works](/en/start/how-it-works/) walks through one full loop in detail.

## Two ways an agent drives it

There are two ways to put a brain behind the rig, covered fully in
[the built-in agent overview](/en/agent/overview/):

- **Bring your own MCP client.** An external client like OpenClaw or Claude Desktop connects to
  the server's `/mcp` endpoint and drives the rig directly.
- **The built-in agent runtime.** PhysiClaw's own engine runs the look→decide→move→check loop
  for you.

## What's actually on the phone?

The "nothing installed" line is *almost* true, and the difference matters — so here it is
straight:

:::note[The honest version]
There's **no App-Store app, no developer mode, and no jailbreak.** But you do prepare the phone
once: turn on **AssistiveTouch**, add three short **iOS Shortcuts** (a tap, a pixel-perfect
screenshot, and clipboard sync), and keep a small **bridge page** open in the browser during a
session. It stays a normal phone you've spent five minutes setting up.
[Prepare the phone](/en/set-up/prepare-the-phone/) covers exactly what and why.
:::

Use a throwaway / burner phone — the runtime's `unlock_phone` enters a hardcoded passcode
(`111111`), so the tool phone should be one you don't mind exposing. See [Safety](/en/safety/)
for the full posture.

## Who it's for, and what you're signing up for

PhysiClaw is for builders and tinkerers — robotics learners and anyone comfortable assembling a
machine and following a wiring diagram. It is a **from-scratch build**, not a ready-made plotter
you bolt a pen onto:

- The X/Y motion is a hand-built **CoreXY** gantry (a two-motor X/Y stage). The stylus is dropped
  onto the screen by a **solenoid** — a push-rod actuator on the Z axis, not a servo or a pen.
- Parts run roughly **$145 USD** (about ¥1046 in China; higher in USD/EU markets).
- Expect **some soldering** for wire splices and a one-time **firmware flash** (FluidNC).
- It's a **multi-day project**: source, cut, and 3D-print parts (vendor lead times run several
  days), then roughly **2–3 hours of assembly**, then flash firmware, then an iterative
  **calibrate-and-tune** phase to get taps landing dead-on.

None of this is exotic — it's a real but achievable weekend-plus build for someone who likes
building things. The full cost and parts list live in [Sourcing](/en/hardware/sourcing/), and the
build itself starts at [Assembly](/en/hardware/assembly/).

:::note[Platform]
macOS is the supported install today; Windows works for most of the flow.
:::
