`physiclaw` CLI 是整个系统的唯一入口：它检查你的机器、运行 MCP 服务器、带你完成一次性设置，
并管理模型、技能和配置。其入口点是 `physiclaw = physiclaw.cli:app`。任意命令加上 `-h` / `--help`
可查看其完整选项，`physiclaw --version` 可打印版本号：

```bash
physiclaw --version
# physiclaw, version 0.0.20
```

顶层命令是 `doctor`、`server`、`status` 和 `uninstall`。分组的子应用是 `setup`、`config`、
`models` 和 `skills`。

## `doctor`

一次 **只读** 的健康检查——它从不改变任何东西。任何东西不工作时，都先用它。

```bash
physiclaw doctor
physiclaw doctor --deep   # 更彻底的检查
```

它能感知服务器：如果有一个 PhysiClaw 服务器正在运行，doctor 会信任该服务器实时的
`/api/status`（服务器已经占着串口和相机，重新探测要么会报告"忙"要么会把它弄坏）。当没有服务器
在运行时，doctor 会主动探测 GRBL 机械臂并枚举相机。它还会把 `0.0.0.0:8048` 这个 LAN 绑定作为
信息性输出展示出来，让你看到服务器在你的网络上是可达的——这是预期行为，不是警告。

| 选项 | 作用 |
| --- | --- |
| `--deep` | 运行更彻底的那组检查 |

## `server`

运行 MCP 服务器——并且，除非另有指示，同时运行内置助理的运行时。

```bash
physiclaw server
```

服务器是基于 streamable-HTTP 的 [FastMCP](https://gofastmcp.com)。启动时它会打印 MCP 端点、
手机桥接 URL 和一个二维码链接，然后打开浏览器中的硬件设置向导。默认绑定为 `host 0.0.0.0`、
`port 8048`，所以端点是 `http://<host>:8048/mcp`——刻意做成 LAN 可达，这样手机桥接才能连上它。

默认情况下，服务器还会在 MCP 服务器旁边派生一个 **助理运行时子进程**——也就是让内置助理能自行
唤醒的那个循环。如果你传入 `--no-runtime`，或者没有配置任何模型（见下文），它会跳过该子进程。

| 选项 | 默认 | 作用 |
| --- | --- | --- |
| `--port` | `8048` | MCP 服务器端口 |
| `--host` | `0.0.0.0` | 绑定地址（LAN 可达，供手机桥接使用） |
| `-v`, `--verbose` | 关 | 详细的调试输出 |
| `--no-runtime` | 关 | 仅运行 MCP 服务器——不派生助理运行时循环 |
| `--warm-start` | 关 | 从已保存的校准包自动连接硬件并跳过向导 |
| `--cam-index` | 校准包中的值 | 为 `--warm-start` 覆盖相机索引 |
| `--no-setup-hardware` | 关 | 启动时不自动打开浏览器设置向导 |
| `--save-tool-calls` | 关 | 把每次 `peek` / `screenshot` 的输出转储到数据目录下 |
| `--save-snapshots` | 关 | 把每一帧原始相机画面转储到数据目录下 |
| `--save-screenshots` | 关 | 把手机自身的每一张原始截图转储到数据目录下 |

已保存的校准包 **仅** 通过 `--warm-start` 恢复；如果校准包不完整，`--warm-start` 会放行继续，
向导照常打开。

:::note[没有模型？服务器照样运行，但助理保持沉睡]
在 **没有配置模型** 的情况下，运行时循环会被跳过——助理无法唤醒——但 MCP 服务器本身仍会运行。
这对硬件设置以及手动或外部的 MCP 工具调用很有用。要唤醒助理，设置一个模型并重启服务器：

```bash
physiclaw models key anthropic                   # 在隐藏提示处粘贴密钥
physiclaw models use anthropic/claude-sonnet-4-6
physiclaw server
```

（或在环境中设置 `PHYSICLAW_MODEL`。）在设置好模型之前，两种驱动模式都会明确报错。
参见[获取 API Key](/zh/set-up/provider/)。
:::

## `status`

一份快速概览快照——视觉模型是否存在、校准状态，以及 jobs 文件。与 `doctor` 不同，它
**不会** 探测硬件，所以可以安全地管道传给 `grep` / `jq`。

```bash
physiclaw status
```

## `setup`

一次性配置，通过三个子命令完成。

### `setup local-vision-model`

下载并转换为 `peek` 和 `screenshot` 提供能力的 OmniParser 图标检测模型。

```bash
physiclaw setup local-vision-model
```

它会下载 `microsoft/OmniParser-v2.0`（`icon_detect/model.pt`）并转换为 ONNX（`imgsz=1280`），
落在 `~/.physiclaw/models/omniparser_icon_detect/model.onnx`。那些重量级依赖（约 500 MB 的
`ultralytics` + `onnx`）在一个用完即弃的 `uv run --with` 环境中运行，所以它们绝不会进入
PhysiClaw 的安装环境——`uv` 必须在你的 PATH 上。传入 `--force` 可在模型已存在时仍强制重新下载。

| 选项 | 作用 |
| --- | --- |
| `--force` | 即使模型已存在，也重新下载并重新转换 |

### `setup hardware`

交互式的机械臂 + 相机校准向导。服务器必须已经在运行。它与浏览器 `/setup-hardware` 向导
逐步对应（使用相同的 `/api/*` 端点）。

```bash
physiclaw setup hardware
```

它走的是十个有序步骤：连接手机桥接、摆好设备、连接机械臂、连接相机、视口偏移、校准机械臂
（**18 个点位**）、校准相机（**15 个圆点**）、验证、核查 AssistiveTouch，以及收尾。

| 选项 | 作用 |
| --- | --- |
| `-a`, `--auto` | 自动模式——手机连接后即布防，然后驱动每一步，仅在出错或需要物理动作的步骤处暂停 |
| `--trace` | 加上可选的边缘描迹精度步骤（8 个边缘点） |
| `--server-url` | 正在运行的服务器 URL（默认 `$PHYSICLAW_SERVER` 或 `http://localhost:8048`） |

:::note[先刷固件]
向控制板刷入 FluidNC 是 `setup hardware` 之前必须通过的前置关卡——在此之前机械臂无法连接。
完整的向导走查见 [校准](/zh/set-up/calibrate/)。
:::

### `setup phone`

教会助理识别屏幕键盘，这样它才能打字。传入一张或多张显示键盘的手机截图——把字母布局和
数字布局都拍下来。

```bash
physiclaw setup phone alpha.png numeric.png
```

它会检测按键框、写出 bbox 预览 PNG（让你能用肉眼核对检测结果），并生成一个 UI 预设 markdown
文件，供助理在打字时读取。少数几个按键槽位会被标成 `???`，必须对照 bbox 预览手工填上。

| 参数 / 选项 | 默认 | 作用 |
| --- | --- | --- |
| `<images...>` | — | 一张或多张能看到键盘的手机截图 |
| `--bbox-dir DIR` | `./keyboard-bbox` | bbox 预览 PNG 的写出位置 |
| `--preset FILE` | `~/.physiclaw/ui-presets/system-keyboard.md` | 生成的 UI 预设 markdown 的写出位置 |

这与 `calibrate-keyboard` 概念是同一套流程。参见 [准备手机](/zh/set-up/prepare-the-phone/)。

## `config`

查看并编辑用户配置文件 `~/.physiclaw/config.toml`。通过 `set` / `unset` 进行的编辑会保留你的
注释和顺序。

| 命令 | 作用 |
| --- | --- |
| `physiclaw config path` | 打印配置文件路径 |
| `physiclaw config show` | 以 TOML 形式转储生效后的配置（API 密钥被遮蔽） |
| `physiclaw config get <section.key>` | 读取一个键，例如 `engine.max_turns` |
| `physiclaw config set <section.key> <value>` | 设置一个键（值会按类型强制转换） |
| `physiclaw config unset <section.key>` | 把一个键还原为其内置默认值 |
| `physiclaw config edit` | 在 `$EDITOR` 中打开文件（不存在时从模板创建） |

设置按以下顺序分层覆盖：**CLI 标志 > 环境变量 > `config.toml` > 内置默认值**。加载时遇到未知键
会明确报错。多数写操作会打印 ``Restart `physiclaw server` to apply.``。

:::caution[别让密钥进入 shell 历史]
`physiclaw config set provider.openai_api_key …` 会把密钥留在你的 shell 历史里。对于 API 密钥，
请优先用 `physiclaw config edit` 或 `physiclaw models key`。
:::
## `models`

列出、查看并切换当前活动模型。模型是一个 `provider/model` 引用（例如
`anthropic/claude-sonnet-4-6`）；PhysiClaw 会从各 provider 的 `/v1/models` 端点 **实时** 发现
其模型目录，而不是内置一份精选清单。

| 命令 | 作用 |
| --- | --- |
| `physiclaw models` | 显示当前活动引用、其来源，以及 provider 的密钥状态 |
| `physiclaw models list [provider]` | 列出已发现的模型（来自本地缓存） |
| `physiclaw models use <provider/model>` | 切换当前活动模型（精确引用；别名：`set`） |
| `physiclaw models key <provider> [value]` | 设置某个 provider 的 API 密钥——省略 `value` 时以隐藏提示输入，随后拉取其实时模型列表 |
| `physiclaw models keys` | 每个 provider 的密钥状态（env / config / 未设置），已遮蔽 |
| `physiclaw models discover <provider>` | 重新拉取某个 provider 的实时模型列表 |

典型的首次运行是两条命令：

```bash
physiclaw models key anthropic          # 在隐藏提示处粘贴密钥
physiclaw models use anthropic/claude-sonnet-4-6
```

进程内有六个 provider——`qwen`、`moonshot`、`openai`、`anthropic`、`google`，以及一个
`deepseek` 占位桩——外加 `claude-code/<model>` 子进程引擎，它会外壳调用 Anthropic 的 `claude`
CLI。`claude-code` 复用 `anthropic` 的密钥和目录，并 **不是** 进程内注册表里的一个独立条目。
（直连的 `anthropic` API provider 与 `claude-code` CLI 子进程是两回事。）参见[获取 API Key](/zh/set-up/provider/)。

## `skills`

从一个 git 仓库源安装、列出并移除技能。技能源是一个带有顶层 `skills/<name>/SKILL.md` 的仓库；
安装会把那棵目录树复制到 `~/.physiclaw/skills/<name>/`。

| 命令 | 作用 |
| --- | --- |
| `physiclaw skills install <name> --from <repo>` | 克隆并安装一个技能（`--ref` 指定分支/标签，`--force` 覆盖） |
| `physiclaw skills list` | 列出已安装的技能及其来源出处 |
| `physiclaw skills uninstall <name>` | 移除一个技能（对用户自建的技能用 `--force`） |

`--from` 接受 `owner/repo` 简写、完整 git URL 或本地路径。用
`physiclaw config set skills.default_source <repo>` 设一个默认源，即可省去 `--from` 标志。
重启服务器以加载新安装的技能。参见[技能](/zh/custom/skills/)。

## `uninstall`

移除 PhysiClaw 的用户 **数据**。不带任何标志时，在删除任何东西前会先提示确认。

```bash
physiclaw uninstall            # 提示确认
physiclaw uninstall --dry-run  # 展示将被移除的内容
```

| 选项 | 作用 |
| --- | --- |
| `--data` / `--all` | 移除整个 `~/.physiclaw` 目录（`--all` 是 `--data` 的别名；两者都传时以 `--data` 为准） |
| `--config` | 仅移除配置文件（保留校准、记忆、模型） |
| `-y`, `--yes` | 跳过确认提示 |
| `--dry-run` | 打印将被移除的内容，但不实际删除 |

:::caution[这只移除数据——不移除二进制文件]
`uninstall` 删除 `~/.physiclaw` 下的文件。它 **不会** 移除 `physiclaw` 命令本身。CLI 由安装它的
工具移除：

```bash
uv tool uninstall physiclaw
```

`uninstall` 会把这条命令作为最后的提醒打印出来，免得你以为已经完事了。
:::
