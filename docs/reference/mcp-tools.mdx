---
title: MCP tools
description: The complete tool surface PhysiClaw exposes to an agent.
---

PhysiClaw exposes six tools over the Model Context Protocol. The agent only ever works in
high-level terms — directions, distances, and gestures — never raw coordinates.

## `screenshot_top`

Parks the stylus out of frame and returns a top-down photo of the screen.

- **Parameters:** none
- **Returns:** image (PNG)

```text
screenshot_top() → <image 1920×1080>
```

## `screenshot_side`

Returns the ~45° side view used to check the stylus tip position.

- **Parameters:** none
- **Returns:** image (PNG)

## `move`

Moves the stylus across the X/Y plane by a direction and a coarse distance.

| Parameter | Type | Values |
| --------- | ---- | ------ |
| `direction` | string | `up`, `down`, `left`, `right`, `up-left`, `up-right`, `down-left`, `down-right` |
| `distance` | string | `small`, `medium`, `large` |

```text
move("down-right", "large")
```

:::note
Working in directions and distances (not pixels) keeps prompts portable across phones and
resolutions — only calibration changes, not the agent's reasoning.
:::

## `tap`

Lowers the stylus to register a touch at the current position, then retracts.

| Parameter | Type | Default | Notes |
| --------- | ---- | ------- | ----- |
| `kind` | string | `single` | `single`, `double`, `long` |

```text
tap()                 # single
tap(kind="long")      # long press
```

## `swipe`

Touches down, drags in a direction, and lifts — for scrolling and flicks.

| Parameter | Type | Values |
| --------- | ---- | ------ |
| `direction` | string | `up`, `down`, `left`, `right` |
| `distance` | string | `small`, `medium`, `large` |

```text
swipe("up", "medium")
```

## `park`

Retracts the stylus fully out of the top camera's frame. Called automatically before every
`screenshot_top`, and available on its own to clear the view.

- **Parameters:** none
- **Returns:** ok

See [Gestures](/en/reference/gestures/) for how `tap` and `swipe` map down to GRBL G-code.
