---
title: 记忆
description: PhysiClaw 如何在多次会话之间记住东西——~/.physiclaw/memory 下的几个纯 Markdown 文件。
---

每次会话都是一次唤醒——PhysiClaw 因某个触发器醒来、做完任务、退出。模型脑子里的东西不会留到下一次唤醒；连续性完全由 `~/.physiclaw/memory/` 下的纯 Markdown 文件重建。

<FileTree>
- ~/.physiclaw/memory/
  - USER.md          你的资料——你来维护，智能体只读
  - memory.md        持久事实，一行一条
  - 2026-06-22.md    当天的活动日志
  - …
</FileTree>

| 文件 | 存放 | 谁来写 |
| --- | --- | --- |
| `USER.md` | 你是谁、你的偏好、硬性规则 | **只有你** |
| `memory.md` | 持久事实，一行一条 | 智能体（`save_memory`） |
| `YYYY-MM-DD.md` | 每天一个只追加的日志 | 智能体（`append_log`） |

每次唤醒时，`USER.md` 和 `memory.md` 很小，会注入系统提示；每日日志可能很大，所以只带上最近几条，需要时再按需拉取更深的窗口（`read_logs`）。预加载条数、默认窗口和上限都可在 `config.toml` 的 `[memory]` 下调整。

`memory.md` 回答「什么是真的？」（你的地址、你爱喝燕麦奶）；每日日志回答「发生了什么？」（`[14:02] 加了 2 份燕麦奶`）。把两者分开，常驻部分就能保持很小，而日志可以随意增长。

:::tip[先填好 USER.md]
一开始就把要点写进 `USER.md`——姓名、地址、常用 App、硬性规则（「绝不从 X 下单」）。它每次唤醒都会读，一份好的资料能让智能体不必反复追问或乱猜。
:::
