---
title: 技能
description: 让 PhysiClaw 学会针对具体 App 的操作流程——技能以文件夹组织，按需渐进加载，保持提示词精简。
---

PhysiClaw 无需任何专门技能即可操作大多数 App。但智能体有时会重复犯同样的错误，或经多次尝试才能完成任务，既消耗 token 又耗费时间。此时可创建一个技能，引导其高效完成特定任务。

## 创建技能

在 `~/.physiclaw/skills/` 下新建一个文件夹（以英文命名，例如京东购物技能命名为 `jd`），并在其中创建名为 `SKILL.md`（全大写）的文件。

<FileTree>
- ~/.physiclaw/skills/
  - jd/
    - SKILL.md
</FileTree>

### SKILL.md 示例

```markdown
---
name: jd
description: 在京东／七鲜购买生鲜蔬果、零食或日用品时使用。
---

# 京东 — 买菜

使用京东七鲜购买生鲜商品。通过 MCP 工具观察屏幕，默认使用 `peek`，尽量不用 `screenshot`。

## 流程

1. 打开「京东」，于首页进入「京东七鲜」。
2. 点击搜索框，输入商品名，进入商品页。
3. 点击「加入购物车」；若弹出规格选择（规格、重量等），选定后点击「确定」。
   - **同一商品页不要重复点击「加入购物车」**，否则会重复添加。
4. 点击购物车图标，核对商品与数量，点击「去结算」。
5. 将商品、数量、价格、收货地址、运费、预计送达时间发送给用户，等待确认。
6. 用户确认后，点击「提交订单／立即支付」。

## 注意：截屏会触发分享弹窗

京东会检测 iOS 截屏并弹出「分享截屏」菜单。点击弹窗外的变暗区域即可关闭。
```

## 最佳实践

**description 决定技能能否被调用。** 它会始终加载进上下文，而技能主体只在被触发后才加载，因此 description 应写明在何种场景、操作哪个 App 时使用，而非泛泛描述功能。

**只记录非显而易见的内容。** 智能体已能操作大多数 App，技能的价值在于补足它尚不了解、或反复出错之处。

**使用祈使句，并说明原因。** 讲清缘由既能让智能体更愿意遵循，也便于举一反三应对同类情况。

**保持精简。** SKILL.md 越短，越省 token。

## 触发技能

给 PhysiClaw 发微信消息时，尽量写清楚用哪个 App，这样更容易触发对应技能。

:::tip[发信息]
帮我在京东买一瓶 400ml 鲜奶
:::
