## 为什么是这个零件，关键在哪

这张表掩盖了一个值得知道的事实：这些零件并非同等重要。下面讲清每一个在做什么，以及哪里可以——哪里不能——将就。

<CardGrid>
  <Card title="臂——唯一的精度件" icon="rocket">
    PhysiClaw 做得好不好，最终都要回到这条臂。它沿 X/Y 把电容笔送到屏幕各处，再靠 Z 把它落下去。点偏了几毫米，是臂的问题，不是 agent 的问题。P25 便宜、对手机大小的工作区来说足够刚、又说标准 G-code——不过 **任何兼容 GRBL 或 FluidNC 的 X/Y 写字机器人都行**。心思花在这里。
  </Card>
  <Card title="Z 轴是一个 solenoid" icon="warning">
    原装 P25 用一个小舵机来抬笔。PhysiClaw 把它换成 **solenoid**（电磁铁）——通电时把电容笔啪地按下去完成点击，断电时抬起。solenoid 给出的是干脆、可重复的敲击，慢吞吞的舵机做不到，这也正是长按和点击在它手里会读出不同效果的原因。[固件](/zh/hardware/firmware/) 通过板子的主轴（spindle）输出来驱动它，而不是走舵机通道。
  </Card>
  <Card title="电容笔必须是软导电纤维" icon="pencil">
    对手机来说这支笔 *就是* 一根指尖，所以它得骗过电容触摸屏。用软的 **导电纤维** 笔头，直径 8–10 mm。硬橡胶或塑料笔头识别不稳，还可能划伤玻璃。这是个 $1.50 的零件，别想太多，但也别换成什么「高精度」硬笔尖。
  </Card>
  <Card title="一台相机，俯拍" icon="laptop">
    一台定焦 USB 摄像头从正上方俯视屏幕——**没有第二台相机**。它干两件事：在电容笔停到一边时，给屏幕一张干净的俯拍读图；以及在点击前，确认电容笔此刻落在哪里。1080p 绰绰有余；定焦更好，这样画面不会来回拉焦。
  </Card>
</CardGrid>

:::tip[替换没问题]
这些没有一个是要特别订制的。写字机器人可以换成任意 GRBL/FluidNC 的 X/Y 机器，摄像头换成任意 1080p 定焦 USB 相机，集线器换成任意 *带供电的* USB 3.0 集线器。硬性要求只有三条：一条刚性的 X/Y 臂、Z 轴上是 solenoid（不是舵机）、以及一个软导电纤维笔头。
:::

## 配角们

清单上其余的东西，只是把那三个重要零件固定到位：

- **鹅颈相机臂**——夹在桌面上，把相机举到屏幕上方 ~25 cm。要金属的，别塑料的：鹅颈一旦发软晃动，每一次点击都会跟着漂。（更多见 [组装](/zh/hardware/assembly/)。）
- **手机座**——一块防滑垫加两个 L 形角块拼成一个角。手机每次都以同样的方式坐进那个角里，这正是让 [标定](/zh/set-up/calibrate/) 能跨次有效的前提。
- **带供电的 USB 集线器**——臂、相机、还有你手机的截图桥接全挂在它上面。用 *带供电的* 集线器，免得板子和相机把笔记本的 USB 口拉到掉压。

:::note[不用焊接，不用自制板]
每个零件买来就是装好的。整个搭建是机械活——夹住、坐稳、插上。唯一的「刷写」是通过 USB 把 [固件](/zh/hardware/firmware/) 加载到控制板上，那是浏览器里点几下，不是接线活。
:::
