龙门搭好了，板子也启动到了 `<Idle|...>`，那为什么它还不能直接用？
因为一台从零搭起的 CoreXY 机器，好不好用全看你亲手调出来的机械状态。
**调试是 [固件](/zh/hardware/firmware/) 和 [软件标定](/zh/set-up/calibrate/) 之间的桥梁**——
而且它绝不是即插即用。

这是最磨人的部分。你要把同步带张紧到目标音高、锁死那颗一松就让一切悄悄崩坏的螺丝、
让导轨松快起来，再把 solenoid 调到敲得干脆、不嗡嗡作响。要做好反复来回的准备。
按下面的顺序逐项调——它顺着运动路径走，从结构一路到触点。

:::note[这一阶段是迭代的——而且会绕回来]
调试和标定互相影响。如果标定之后，命令的移动尺寸不对、或者点击落不到目标上，
解决办法通常还得回到这里（同步带张力、紧定螺丝、或 solenoid 时序）。别指望一次到位。
:::

## 1. 同步带张力

CoreXY 用两条同步带（A 和 B）共同分担两个轴。**两条等张比那个具体数字更重要**——
如果 A 和 B 张力不一致，斜向移动就会走偏，点击也会落空。先把两条带调到同一个音高，
再把这个音高调对。

用手机上的**调音 app**（任何吉他/弦乐调音器都行）来测。像拨吉他弦那样拨一段同步带，读出频率。

<Steps>

1. **把滑座停到 220 mm 位置。** 同步带的音高取决于自由段的长度，所以每次都要从同一个滑座位置量，
   保持一致。

2. **拨弦并读数。** 拨那段较长的同步带，在调音 app 里读频率。目标是 **75 Hz**。

3. **调整后再拨。** 在皮带夹处拧紧或放松，然后再拨一次。重复直到这一段读数约 75 Hz。

4. **再调第二条——然后两条复查。** 把 A 和 B 都张紧到约 75 Hz，并确认两者一致。
   调一条往往会带动另一条，所以来回调到两条读数相同为止。

</Steps>

:::caution[A/B 等张才是重点]
一条 75 Hz、一条 90 Hz 的同步带，走直线时滑座没问题，斜向移动却会走偏——
而你直到标定看起来歪了才会发现。把它们调一致。
:::

## 2. 驱动同步轮的紧定螺丝——无声的杀手

这是**本页最重要的一处修正。** 每个 NEMA 17 都通过一颗小小的紧定螺丝，把一个
**GT2 20 齿驱动同步轮**夹在 **5 mm 电机轴**上，再带动同步带。如果那颗紧定螺丝稍有松动，
同步轮就会在负载下打滑空转，**机械臂会悄无声息地丢失位置**——没有报错、没有报警，
只是点击慢慢偏离目标。这是**用户反馈中排第一的故障。**

一劳永逸地修好它：

<Steps>

1. **让紧定螺丝顶在轴的平面上。** 电机轴上有一个加工出来的平面。转动同步轮，让紧定螺丝
   正好顶在那个平面上，而不是顶在圆弧面上。

2. **拧紧。** 用六角扳手拧到牢固扎实——这颗螺丝承受着每一次移动。

3. **用 Loctite 243 锁固。** 在螺纹上滴一滴 **Loctite 243**，防止它被振动松脱。
   这是本搭建中针对紧定螺丝有据可循的做法——这里要用，任何其他会松动的紧定螺丝也照此处理。

</Steps>

:::tip[如果之后位置漂移，先怀疑这里]
同步轮打滑看起来和标定误差一模一样——移动走得偏短或偏移。
重新标定之前，先复查这两颗紧定螺丝。
:::

## 3. 导轨——顺滑、均匀的行程

运动跑在 **3 根 MGN9H 直线导轨**上（2 根 240 mm 用于 Y，1 根 150 mm 用于 X），配 H 型滑座。
它们**出厂时裹着运输防锈油**，那层油会粘附细屑，而且也不是合适的润滑剂。

<Steps>

1. **擦掉运输防锈油。** 把每根导轨和滑座都清理干净。

2. **重新涂锂基润滑脂（NLGI 2）。** 在导轨上抹一层薄而均匀的油膜。

3. **校准导轨，然后感受行程。** 用手把每个滑座从一端推到另一端。你要感受的是
   **顺滑、均匀、低摩擦、没有任何卡滞的行程。** 出现卡点说明导轨没对正、或被过度约束——
   松开它的固定座，轻轻挪一下，重新拧紧，再感受一遍。

</Steps>

:::tip[校准要有耐心]
导轨校准是个迭代过程。松开、轻挪、重新拧紧、推动、重复，直到滑座在整段长度上都能自由滑过。
这很正常——不要硬推一根跟你较劲的导轨。
:::

## 4. solenoid——让一次点击被识别

Z/触点执行器是一个 **12 V 0.8 A 推拉式 solenoid**，通过主轴 PWM 输出来驱动，而不是舵机。
一次点击是一记**击打并保持**的电磁铁脉冲：先来一记满功率击打把触控笔啪地按下去，短暂稳定，
再降到较低的保持电流以免线圈过热，最后释放。

### 前提：主轴模式

要让 solenoid 能响应 S 值，板子必须处于主轴/激光模式：**`$32=0`**。

:::note[这由配置管，别跟固件硬碰]
这是一个运行时的 `$` 设置，但在 PhysiClaw 上它由 FluidNC 配置
（[`fluidnc_config.yml`](https://github.com/physiclaw/PhysiClaw/blob/main/firmware/fluidnc_config.yml)）
管辖，而非手敲的 `$` 命令——对于配置所管的设置，固件会**拒绝** `$` 写入。
如果 `$32` 不是 `0`，就在 YAML 里改好重新保存，而不是在控制台改。详见
[固件](/zh/hardware/firmware/)。
:::

### 标准的击打并保持时序

这些数字是权威依据。[手势参考](/zh/reference/gestures/) 对它们做了定义；请原样引用。

| 参数 | 值 | 含义 |
| --- | --- | --- |
| `HIT_S` | **1000** | 峰值击打功率 |
| `SETTLE_MS` | **80** | 击打后的停留时间 |
| `HOLD_S` | **750** | 持续保持（必须 **< `HIT_S`**） |
| `RELEASE_MS` | **200** | 释放后让弹簧复位的停留时间 |
| `DOUBLE_TAP_GAP_MS` | **100** | 双击两次之间的间隔 |

发射序列：

```gcode
M3 S1000    ; strike — stylus snaps down (HIT_S)
G4 P0.08    ; settle 80 ms (SETTLE_MS)
M3 S750     ; drop to hold current (HOLD_S, < HIT_S so it doesn't overheat)
M5          ; release — spring lifts the stylus
G4 P0.2     ; 200 ms spring-clear dwell (RELEASE_MS)
```

:::caution[两个过时的数字会让你得到一记软弱或嗡响的点击]
旧文案与上面的值相矛盾。**以本页和 [reference/gestures](/zh/reference/gestures/) 为准，
不要信那两处：**

- [固件](/zh/hardware/firmware/) 页里那段旧代码用的是 `G4 P0.05`（**50 ms** 停留）。
  那是**过时的——用 80 ms。**
- YAML 注释里那句 "S300, tune from S150 up" 是**过时的——保持值是 S750。**

照抄旧代码的人，得到的要么是按不到位的击打，要么是嗡嗡作响的线圈。
:::

### 如果线圈嗡嗡作响

听得见的嗡声说明 PWM 纹波让线圈在振动。两种办法任选其一：

- **在线圈两端并联一个 470 µF 电容**，或者
- **把 PWM 频率提高到 25 kHz**（高过人耳可闻范围）。

### 如果点击仍然不被识别

点击可能在机械上确实发出了，却依然不在屏幕上被识别。电容触控笔的笔尖必须接到一个
**已验证的大地接地**——比如一台已接地的 PC 机箱。**绝不能接墙插，绝不能接控制板的 GND。**
这一步在 [组装](/zh/hardware/assembly/) 时就设好了，但当一记干净的击打却没有产生触摸时，
它是第一件要重新确认的事。
