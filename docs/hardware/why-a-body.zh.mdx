---
title: 为什么 PhysiClaw 需要一个身体
description: 数据之墙真实存在，没有任何软件花招翻得过去。于是我们绕道而行，走那扇大科技公司锁不上的门。
---

个人助手，为什么还需要一个机器人？它既然已经住在你的手机里，难道不能直接跟那些应用打交道吗？

它做不到。而它之所以做不到，恰恰就是整个问题的症结。

你所依赖的每一个服务——Amazon、淘宝、Uber Eats、美团——都躲在一堵墙后面。墙里的数据，是它们最值钱的资产，也守得最紧。它们没有任何理由为你的助手开一扇门，却有一万个理由把门关死。而一个开放的 API——能让你的助手在它们的服务里自由进出的那种——正是它们最不可能推出的。

所以，再聪明的软件花招也救不了你。任何想从正门伸手进去的 agent，都会跟所有人一样，撞上同一堵墙。在可预见的未来，没有谁会把那些主宰你日常生活的服务，免费、开放地交到你手上。

既然穿不过这堵墙，那就绕过去——从那扇他们锁不上的门走。

那扇门，就是屏幕。就是人用手指去点的那个界面。他们可以关掉自己的 API，却没法把应用本身也对用户关上。那些你每天都在触碰的按钮、菜单、界面，必须一直开着——应用一关，生意也就没了。

所以，PhysiClaw 走的正是这扇门。它像你一样操作你的手机：用相机当眼睛，用触控笔当手指，用屏幕本身当 API。凡是你能用手做到的，它也用手去做——通过一模一样的界面，以一模一样的方式。

这是有代价的，我们不会假装没有。同一个任务，软件几毫秒就能搞定，PhysiClaw 却要花上几分钟——它得看一眼、移动、点击，再看一眼。它更慢。慢得多。

但它换来了任何 API 都给不了的东西：一种谁也无法收回的访问权。PhysiClaw 走得慢，可换来的是——它哪儿都去得了。它用速度，换来了自由。

这，就是 PhysiClaw 骨子里的那笔交易。那堵墙，是用软件砌成的——权限、令牌、服务条款。而一只手不是。你没法阻止一只手去触碰屏幕。

数据之墙，打得破——不靠代码，靠硬件。
