在 PhysiClaw 能操控手机之前，手机需要一次大约五分钟、每部只做一次的设置：打开
**AssistiveTouch**、添加**三个 iOS 快捷指令**，并在 Safari 里打开**桥接页面（bridge page）**。
这些都不会安装任何 app、开发者描述文件，也不涉及任何越狱——全都是 iOS 自带的功能。但它正是
「一支触控笔戳玻璃」和「一部 PhysiClaw 能完整*看见*（像素级精确的截图）并*操作*（粘贴文本）的
手机」之间的区别。

下面的步骤基于 iOS。整个流程假定[服务已在运行](/zh/set-up/install/)，因为快捷指令和桥接页面要
通过你的局域网和它通信。

:::caution[用一部专用淘汰机]
PhysiClaw 拿到的访问权，和一个攥着你已解锁手机的人一样多。请把这套设置做在一部专用的工具机上，
而不是你的主力机。你会给它设一个用完即弃的 `111111` 密码，还会有一支接地的触控笔整天戳它的屏幕。
[安全](/zh/safety/) 会详细解释完整的安全姿态，以及为什么这很重要。
:::

## 每一块为什么存在

让其余部分变得显而易见的直觉是：**触控笔只能戳玻璃。** 它没有办法「按一个键」、触发某个系统操作，
或读取剪贴板。所以有三个问题要解决，每一个都对应设置中的一块。

| 这一块 | 为什么存在 |
| --- | --- |
| **AssistiveTouch** | 屏幕上那个唯一可供触控笔物理戳到、用来触发快捷指令的悬浮按钮。没有它，触控笔根本没办法触发任何系统操作。 |
| **3 个 iOS 快捷指令** | 单纯戳一下做不到的能力：捕获一张像素级精确的截图并上传，以及把排队的文本取到剪贴板上以便粘贴。 |
| **桥接页面** | 一个整个会话期间一直开着的 Safari 页面。它画出俯拍摄像头会自动识别的彩色角标，并把文本在剪贴板两端中转。 |

按依赖顺序来搭建：**桥接页面 → 三个快捷指令 → 绑定到 AssistiveTouch → 密码 → 验证。**

## 1. 打开桥接页面

桥接（bridge）是一个 Safari 页面，手机在整个会话期间一直开着它。它干两件事：

- 它画出**彩色的角标**——每个角落一小簇红/绿/蓝/黄（RGBY）色块——供俯拍摄像头自动识别。
  [校准](/zh/set-up/calibrate/) 在连接摄像头那一步用它们来挑出俯拍摄像头并定位屏幕，而蓝/红
  标记还会在摄像头校准那一步固定摄像头的旋转方向。
- 它接收服务排队发来的文本并中转到剪贴板，这样助理就能*粘贴*一段消息，而不必一个字母一个字母地戳出来。

桥接 URL 是这种形态，其中 `8048` 是默认的服务端口：

```text
http://<host>:8048/bridge
```

扫描服务打印出的二维码来配对手机：

<Steps>

1. 在 Mac 上，打开服务打印出来的那个二维码页面：`http://localhost:8048/api/bridge/qr`。

2. 用手机的**相机** app 扫它，在 **Safari** 里打开链接。

3. 页面应当加载出来并显示 **PhysiClaw**。让它停在**前台**，并保持屏幕**常亮**——屏幕一旦睡眠，
   摄像头就找不到那些标记了。

</Steps>

:::note[离线？手动输入 URL]
二维码图像是用从 jsDelivr CDN 加载的库渲染的。如果 Mac 处于离线状态，二维码页面就画不出码，转而
显示一个**手动输入 URL 的兜底方案**——在手机上打开 Safari，手动输入桥接 URL。
:::

### 用哪个地址：优先 `<name>.local`

你接下来要搭建的快捷指令指向的是同一个服务。**强烈建议用 `<name>.local` 这种 mDNS 地址，而不是
写死的 `<lan-ip>`：** 写死的 IP 会在路由器（DHCP）下次分配新地址时失效，而 `<name>.local` 在同一个
Wi-Fi 下即使 IP 变动也依然有效。

要拿到正确的地址：

- 运行 **phone-setup 技能**，它会打印出推荐的 `<name>.local` URL（以及 IP 兜底地址），或者
- 从服务打印出的二维码 URL 里读出来。

`.local` 名字就是你的 Mac 当前的名字——本页不会替你重命名主机。如果你想要一个更稳定或更独特的名字，
自己改即可（例如 `sudo scutil --set LocalHostName "new-name"`），然后用新的 `<name>.local`。

## 2. 搭建三个 iOS 快捷指令

每个快捷指令都只是一两个操作。在**快捷指令** app 里搭建它们：点 **+** 新建一个，添加操作，再重命名。
把 `HOST` 换成你的桥接主机——也就是 `host:port` 部分，例如 `name.local:8048`。

<Steps>

1. **PhysiClaw Tap** ——*截一张屏。*
   添加 **截屏（Take Screenshot）** 操作。把快捷指令重命名为 `PhysiClaw Tap`。这是整条管线里负责
   捕获的那一半；它会往「照片」里放一张新鲜的截图。

2. **PhysiClaw Screenshot** ——*上传最新的截图。*
   添加 **获取最新的截屏（Get Latest Screenshots）**，再加 **获取 URL 内容（Get Contents of
   URL）**。设置：
   - URL：`http://HOST/api/bridge/screenshot`
   - **显示更多** → 方法选 **POST**，请求正文选 **文件**，文件 → **Screenshots** 变量

   重命名为 `PhysiClaw Screenshot`。这会把像素级精确的捕获上传到服务。

3. **PhysiClaw Clipboard** ——*同步剪贴板。*
   添加 **获取 URL 内容（Get Contents of URL）**，URL 设为 `http://HOST/api/bridge/clipboard`
   （方法 **GET**），再加 **拷贝到剪贴板（Copy to Clipboard）**，把它的输入设为 **URL 的内容**。
   重命名为 `PhysiClaw Clipboard`。这会把服务排好队的文本取下来、放到剪贴板上，随时可以粘贴。

</Steps>

## 3. 把快捷指令接到 AssistiveTouch 上

AssistiveTouch 是 iOS 屏幕上那个悬浮按钮——触控笔戳的是*它*，由它来触发快捷指令。打开它，并绑定
三个手势。

<Steps>

1. **设置 → 辅助功能 → 触控 → AssistiveTouch** → **打开**。

2. 在 **自定操作** 下，设置：
   - **单点（Single-Tap）** → 快捷指令 → **PhysiClaw Tap**
   - **轻点两下（Double-Tap）** → 快捷指令 → **PhysiClaw Screenshot**
   - **长按（Long Press）** → 快捷指令 → **PhysiClaw Clipboard**

</Steps>

这套「手势到快捷指令」的映射正是运行时所依赖的。[`screenshot` 工具](/zh/reference/mcp-tools/) 会在
AssistiveTouch 按钮上做一次**先单点再双击**来捕获并上传，而 `send_to_clipboard` 及粘贴流程则用
**长按**来取剪贴板。

:::tip[摆放位置很重要]
每个手势都会持有一把锁，并且**拒绝任何越过 AssistiveTouch 按钮的移动**，触控笔在两次操作之间还会
自动停泊到屏幕外。把 AssistiveTouch 按钮放在触控笔不必从它上面划过的地方——屏幕的某个角落就很合适。
关于物理触控笔的移动如何映射成点击和滑动，见[手势](/zh/reference/gestures/)。
:::

## 4. 把密码设为 `111111`

PhysiClaw 能自己解锁手机，但它只知道一个密码：`111111`。运行时的
[`unlock_phone` 工具](/zh/reference/mcp-tools/) 会唤醒手机、上滑、对小键盘做 OCR，并输入这唯一一个
**写死的**密码。真实密码是故意从不存储的——如果它躺在配置文件里或被打进键盘逻辑，就可能从日志或
git 历史里泄露。

把工具机的密码设为 `111111`（**设置 → 面容 ID 与密码**），并且不要在这部手机上留任何你不愿交给
陌生人的东西。`111111` 是给一部用完即弃的手机用的、用完即弃的密码。

:::note[这是运行时要求，不是校准步骤]
密码只在运行时需要解锁手机时才有意义——它**不是**[校准向导](/zh/set-up/calibrate/) 的一部分。
如果你宁愿 PhysiClaw 永远不去解锁手机，那就改为关掉自动锁定：**设置 → 显示与亮度 → 自动锁定 →
永不**，让手机保持解锁。代价是屏幕长亮会更快地老化。
:::

## 5. 验证它能工作

在继续之前，确认捕获与上传这条路径确实抵达了服务。

<Steps>

1. 在 AssistiveTouch 按钮上点**一下**——这会截一张屏。

2. 点**两下**（双击）——这会上传最新的截图。

3. 大约 **25 秒**内，服务端应当落下一个新文件。

</Steps>

如果什么都没到：

- 在快捷指令 app 里**手动运行该快捷指令**（点 **▶**），把它遇到的任何错误暴露出来。
- 在 Mac 上 `ping <name>.local`。如果失败，说明你的网络屏蔽了 mDNS——把**两个**快捷指令 URL
  （`PhysiClaw Screenshot` 和 `PhysiClaw Clipboard`）都换成**裸 IP** 兜底地址。
- 确保「照片」里**至少已有一张截图**，这样*获取最新的截屏*才有东西可上传。

## 下一步：校准

打开了 AssistiveTouch、搭好了三个快捷指令、开着桥接页面之后，手机就准备好了。把它交给
[校准向导](/zh/set-up/calibrate/) ——它的 **验证 AssistiveTouch** 那一步会重新跑一遍单点、双击和
长按，在装置开始驱动之前端到端确认三个快捷指令都正常。
