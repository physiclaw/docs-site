---
title: Get an API key
description: Pick a model provider, grab an API key, and add it to PhysiClaw — OpenAI, Anthropic, Google, Qwen, or Kimi.
---

Calibration needs no model, but to let the agent actually drive the phone you need one model from one provider. Create an account, generate an API key, then add it to PhysiClaw.

| Provider | Where to get the key | Provider id |
| --- | --- | --- |
| **OpenAI** | [platform.openai.com/api-keys](https://platform.openai.com/api-keys) | `openai` |
| **Anthropic** (Claude) | [console.anthropic.com/settings/keys](https://console.anthropic.com/settings/keys) | `anthropic` |
| **Google** (Gemini) | [aistudio.google.com/apikey](https://aistudio.google.com/apikey) | `google` |
| **Qwen** (通义千问) | [bailian.console.aliyun.com](https://bailian.console.aliyun.com) | `qwen` |
| **Kimi** (Moonshot) | [platform.moonshot.cn](https://platform.moonshot.cn/console/api-keys) | `moonshot` |

:::tip[Copy the key right away]
Most consoles show the full key only once. Copy it immediately and keep it somewhere safe.
:::

Then add the key and pick a model:

```bash
physiclaw models key <provider>      # paste the key when prompted
physiclaw models use <provider/model>
```

For example: `physiclaw models key anthropic`, then `physiclaw models use anthropic/claude-sonnet-4-6`.

See [Models](/en/agent/models/) for the full list and how model selection works.
