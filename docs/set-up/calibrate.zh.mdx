PhysiClaw 用摄像头观察屏幕，再用触控笔点击屏幕。为此，它需要建立两个映射：

一个是从**手机屏幕**到**摄像头画面**——也就是搞清楚「屏幕上的某个按钮，出现在摄像头画面里的哪个位置」。

另一个是从**手机屏幕**到**机械臂移动平面**——也就是算出「要点到这个按钮，触控笔需要移动多远」。

有了这两个映射，PhysiClaw 才能「眼手合一」：看到画面后，准确点到正确的位置。

## 摄像头

开始校准前，先在电脑上打开一个相机预览应用，查看摄像头拍到的画面，并调整摄像头角度，让手机屏幕：

- **尽量占满画面**
- **保持平整**
- **不倾斜**

可用来查看摄像头画面的应用：

- **macOS** —— Photo Booth
- **Windows** —— 相机
- **Linux** —— Snapshot（或 Cheese）

:::caution[把手机垫平]
手机的后置摄像头凸起会让机身无法平放。在手机下方垫上裁成合适形状的薄卡纸或软垫，把它垫平，确保屏幕保持水平、不晃动。
:::

## 设置

启动服务：

<Tabs syncKey="os">
<TabItem label="macOS">

```bash
physiclaw
```

第一次读取摄像头时，macOS 会弹出授权请求，点击**允许**。也可在**系统设置 → 隐私与安全性 → 摄像头**里重新设置。

</TabItem>
<TabItem label="Windows">

```powershell
physiclaw
```

</TabItem>
<TabItem label="Linux">

读取摄像头和访问串口分别需要 `video` 和 `dialout` 用户组。把自己加入这两个组，然后**注销并重新登录**：

```bash
sudo usermod -aG dialout,video $USER
```

（Arch 系发行版的串口组是 `uucp`，不是 `dialout`。）

然后启动服务：

```bash
physiclaw
```

</TabItem>
</Tabs>

它会在浏览器里自动打开一个设置向导，一步步带你完成校准。

### 自动模式

向导支持自动运行。在**自动模式**下，它会替你完成每一个校准步骤。

如果你之前已经完成过校准，没有移动过硬件，可以开启自动模式。

## 首次运行

第一次运行，在校准完成后，PhysiClaw 会花几分钟学习手机的屏幕布局——聊天软件的输入框、发送按钮和屏幕键盘的位置。这一步全自动，你无需操作。

请确保**微信**已固定在屏幕底部栏（见[准备手机](/zh/set-up/prepare-the-phone/)），PhysiClaw 就能找到并打开它。学习完成后，就可以开始给它派任务了。

:::tip[换了手机？]
运行 `physiclaw reset` 清除校准数据和已学习的手机布局，再重启 PhysiClaw，它会重新校准并重新学习新手机的界面。
:::
